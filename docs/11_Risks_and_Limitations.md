# AI Response Quality Evaluator

# 11. Risks and Limitations

## Introduction

Every software project has some limitations and challenges. Identifying these limitations helps in planning future improvements.

---

# Project Limitations

The project depends on the quality of the reference knowledge base.

If the reference information is incomplete, the evaluation may also become less accurate.

The quality of embeddings also affects the retrieval process.

The project currently focuses only on evaluating AI-generated responses and does not generate responses itself.

---

# Possible Risks

Sometimes relevant reference documents may not be retrieved correctly.

Large datasets may increase the processing time.

Incorrect or incomplete user input can also affect the evaluation result.

---

# Challenges

Understanding different LLM evaluation techniques required detailed research.

Designing multiple evaluation agents and connecting them through one workflow was another important challenge.

Building a reliable RAG pipeline also requires proper preprocessing and embedding generation.

---

# Future Improvements

The project can be improved by using larger datasets, better embedding models, and more advanced evaluation techniques.

Additional evaluation agents can also be added in future versions.

---

# Conclusion

Although the project has some limitations, the modular architecture makes it easy to improve the system in future updates.
