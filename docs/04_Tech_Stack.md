# AI Response Quality Evaluator

# 4. Tech Stack

## Introduction

Choosing the right technologies is very important for any project. I selected the tools and technologies according to the requirements of my project. The main focus was to use technologies that are easy to understand, easy to integrate, and suitable for building a multi-agent AI evaluation system.

---

# 1. Python

Python is the main programming language used in this project. It is simple, easy to learn, and provides many libraries for Artificial Intelligence and Machine Learning projects. All the project modules will be developed using Python.

---

# 2. Streamlit

Streamlit will be used to build the user interface of the project. It allows users to enter the question, AI response, and reference answer. It will also display the final evaluation report in a simple dashboard.

---

# 3. LangChain

LangChain will be used to build the multi-agent workflow. It helps in managing different evaluation agents and allows them to work together in an organized manner.

---

# 4. ChromaDB

ChromaDB will be used as the vector database. It stores embeddings generated from the reference datasets and retrieves the most relevant information whenever required during evaluation.

---

# 5. Hugging Face Datasets

Public datasets like TruthfulQA and SQuAD will be collected from Hugging Face. These datasets will be used to build the reference knowledge base for the RAG pipeline.

---

# 6. Sentence Transformers

Sentence Transformers will generate embeddings for the reference documents. These embeddings help ChromaDB retrieve similar information during evaluation.

---

# 7. Git and GitHub

Git will be used for version control, while GitHub will be used to store the project repository and maintain all project files.

---

# Conclusion

The selected technologies are suitable for building a modular AI response evaluation system. They are open source, easy to use, and support future improvements in the project.
